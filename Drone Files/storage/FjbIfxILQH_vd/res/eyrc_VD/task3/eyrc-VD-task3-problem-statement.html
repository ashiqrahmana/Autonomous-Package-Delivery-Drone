Robotics Competition
2020-21

eYRC 2020-21: Vitaraṇa Drone (VD)

Problem Statement

The aim of this task is to detect the landing markers present in the general vicinity of a given GPS coordinate. The GPS coordinate of the landing marker will NOT be given, instead the coordinate will point somewhere around the marker. You have to hover at certain height from the given coordinates and scan for the landing marker by applying image processing techniques and design an algorithm to take the eDrone above the landing marker and eventually land on it.

Procedure

We will be using the same world as the last task but with the markers and drone position changed, use the launch file task_3.launch to launch the relevant world and complete this task.

  1. Navigation:

    The GPS co-ordinates of the building are as follows:

    • Building 1: lat: 18.9990965928, long: 72.0000664814, alt: 10.75
    • Building 2: lat: 18.9990965925, long: 71.9999050292, alt: 22.2
    • Building 3: lat: 18.9993675932, long: 72.0000569892, alt: 10.7

    You can detect them in any order you wish. You have to land on the last marker to end the run.

    NOTE: These are not the marker coordinates. You have to arrive and hover exactly 1 metre above this coordinate before commencing your detection (This is just for the present task, future approaches will be up to you, threshold for this task is ±0.2m in all directions.

    You may proceed with the detection phase however you want after reaching the above mentioned spot. Make sure to follow the post-detection steps.

  2. Marker Detection:

    • After reaching the coordinates provided, you have to begin your search for the landing marker in the vicinity using the bottom facing camera on the eDrone, you can develop any kind of search algorithm you like.
    • You may use any sort of image processing algorithm to detect and localise the markers.
    • As you know, OpenCV, a popular image processing library, comes preinstalled with ROS.
    • Storing precise marker locations in memory is prohibited since it beats the purpose of the task.
    • We have presented a generalised robust method to detect any sort of distinctive shape in the learning resources. It uses cascaded classifiers, some of you might have heard of Haar Cascades.
    • You can find it here: Cascade Classifiers
    • You may use it or any other method, in the one we present, you can easily swap out and detect multiple types of of distinctive shapes, i.e. landing markers of different styles, hence we highly recommend you try it.
    • Remember, the end output from the detection phase should be that you have the pixel coordinates of the centre of the marker detected w.r.t the camera frame of eDrone. You will use this centre point of the marker to navigate the eDrone above it.

  3. Post marker detection steps:

    • After you have detected a marker you have to go and hover above it at your desired z coordinate (you can go up or down).

    • Meanwhile you have to compute the horizontal distance in meters from the drone to the marker using the data from the image. We need the distance to the centre, although the centre does not need to be pixel or sub-pixel level precise.

    • To do so, we will use the following formula. First, we will calculate the focal length of the camera.

      focal_length = (img_width/2)/tan(hfov_rad/2) where,

      image width in our case is 400 pixels

      hfov_rad stands for the horizontal field of view of the camera in radians, in our case it is 1.3962634

    • Then, X and Y distance of marker centre from the eDrone in metres are given by centre_x_pixel*Z_m/focal_length

      Note that this depends upon the orientation of eDrone w.r.t. the world.

    • The Z_m stands for the vertical distance in metres and can be obtained as you know both the landing marker altitude and your present alt, alternatively you can also use the downward pointing range sensor.

    • You need to publish these X and Y values in the global coordinate frame, i.e. X is global X and Y is the global Y, both with the correct signs. We instruct you to detect and publish your results at a frequency of 1 Hz on the topic /edrone/marker_data.

    • The topic /edrone/marker_data uses the custom ROS message MarkerData which has already been created in the updated package of vitarana_drone. The message structure can be seen by typing 

      rosmsg show vitarana_drone/MarkerData
      Fig 1: Rosmsg show MarkerData

    • Floating point data of the X and Y values should be published on err_x_m and err_y_m

      • Integer data of the Building ID (use the same numbers 1, 2, 3 as above) of the marker you are presently in the process of detecting/hovering; on marker_id
      • Example: Say there is a marker on Building 1 at [1, 2, 2] and your quad is at [3.3, 4, 6], you should follow the procedure above of processing your pixel coordinate centres and obtain, err_x_m = -2.3 and err_y_m = 2 and marker_id = 1

    • You should go towards the marker maintaining the z coordinate by minimising your x and y distance. The threshold in metres should be below 0.2 metres before moving to the next building's terrace.

    • Indicate that you are moving to the next building by changing and publishing the marker_id appropriately. Do this as soon as you are within the threshold mentioned in the above point. You do not need to do this for the last building, instead, land on the last marker you chose to detect and visit.

      • You may verify these using the static world for convenience.

Summary Steps:

  1. Indicate where you're going by publishing Building number in integer form
  2. Arrive at any of the GPS coordinates given above +1 m vertical (threshold 0.2m)
  3. Commence detection stage, i.e. search for the marker
  4. Once detected, keep publishing X & Y error in metres at 1Hz using the formula mentioned above until both are reduced to below 0.2 i.e. go towards the marker while publishing.
  5. If not last marker, go to Step 1 and repeat, else, land on the last landing marker.

All the best!
