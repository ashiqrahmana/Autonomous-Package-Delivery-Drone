eYRC 2020-21: Vitaraṇa Drone (VD)

Task 2C

Navigation

This task will merge tasks 2A and 2B with obstacle avoidance and path planning. Use the onboard rangefinder sensors on the eDrone to detect obstacles and navigate around them. This document, contains relevant rules and information on how to create one. While there is no limitation on script structure, it is recommended that you merge the navigation logic in your position control script as we think that will be the easiest way forward.

Task

  • After you complete the navigation, the final task is to use it to navigate to picking up a box, reading the QR code on it to know the destination and delivering it to the destination.

  • The Gazebo world for the task is prepared. To launch it type

    roslaunch vitarana_drone task_2.launch
Problem statement:

  • The eDrone starts from the start position i.e. latitude: 19.0009248718 longitude: 71.9998318945 altitude: 22.16, this position is already entered in the relevant world file and the eDrone will spawn at this location itself and it will be in view.

  • It is mandatory that the eDrone start from here during your submission recording.

  • The eDrone has to go to latitude: 19.0007046575 longitude: 71.9998955286 altitude: 22.1599967919, there will be a box with a QR code on that position, the QR code will contain the destination of the box, the eDrone should scan the QR code.

  • Pick up the box and deliver it to the destination while navigating the airspace.

  • Gently drop the box on the destination platform, end your recording after doing so.

NOTE:

This is the cumulative submission of Task 2, read the Submission Instructions for more details.
