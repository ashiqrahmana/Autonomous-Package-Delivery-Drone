eYRC 2020-21: Vitaraṇa Drone (VD)

Problem Statement

The aim of this task is to write a scheduler algorithm to plan the order of jobs (delivery or return) such that the the earnings of eDrone is maximum in the given time limit.

Details

  • The eDrone will spawn on the roof of a warehouse.

  • The warehouse will have two different grids, one for deliveries on which the parcel boxes are placed and one dor returns on which the parcel boxes are to be dropped . The columns of delivery grid cells are named A, B, C and the columns of return grid cells are named X, Y, Z. The GPS coordinate of the cell A1 is lat: 18.9998102845 long: 72.000142461 alt: 16.757981 and that of X1 is lat: 18.9999367615 long: 72.000142461 alt: 16.757981. The size of each cell is 1.5m x 1.5m which translates approximately to 0.000013552 in latitude and 0.000014245 in longitude, so one can calculate the GPS coordinates of any cell on the grids with this information.

  • The list of deliveries and returns is provided in the manifest.csv in the scripts folder of vitarana_drone ROS package.

  • The .csv will have the structure explained in this section of Rulebook.

    Note: In case of returns, the origin coordinates are the exact coordinates of the parcel box, and the destination is the cell number on the warehouse grid where the parcel box has to be dropped. Whereas in case of deliveries, the origin is the cell number on the warehouse grid and the destination coordinates are the building coordinates and you need to scan for the landing marker as done in previous tasks.

  • The eDrone can choose to complete any number of jobs in any order, analyzing the cost (distance and time) and the respective earning for that job.

  • You need to write a scheduler algorithm to choose and schedule jobs such that most of the time the eDrone is occupied (completing a job).

  • For example, lets say the eDrone is on its way to deliver a parcel to a destination and there is a return parcel from a building just near to it, so an efficient algorithm will schedule the return just after the delivery so that the eDrone has to travel less distance for the next job, wasting less time and maximizing the efficiency.

  • The more efficiently you choose and schedule jobs (considering their cost and its earning), the more will be the total earning in the given time period. Note: Remember, completing more jobs doesn't mean more earnings, keep in mind that the earnings also increase as the distance of the delivery increases, so design the scheduler wisely.

  • Since all the information is provided in the manifest.csv, one can calculate the distance between two locations and plan the entire job order accordingly.

  • You can write the scheduler algorithm in a separate python script and make a separate rosnode or you can combine the algorithm in existing scripts.

  • The scheduler algorithm should import the manifest.csv file containing the details of deliveries and returns and plan the entire order i.e. sequence before the eDrone takes off.

  • You should output another .csv file titled sequenced_manifest.csv that has the same structure as the manifest.csv but the rows are ordered i.e. sequenced according to the order in which the eDrone will transport the parcels. An example is given in the table below.

    NOTE: The order or sequence of the jobs should NOT be hard coded, if found, team will be disqualified.

  • The earnings of the eDrone will be the score for this task and the formula for earning can be referred from the Rulebook.
